\## SLG-GAME

一款基于 C++ 和 SFML 开发的简易SLG游戏。



\## 技术栈

C++17

SFML 3.0（图形渲染）

MySQL / SQLite（数据持久化）

JSON（序列化存档）



\## 核心功能

地图网格渲染与 Chunk 分块优化

县、郡、州三级管理体系

双数据库引擎适配

完整存档系统（JSON 序列化）

数据驱动设计（配置文件控制游戏数值）



\## 技术亮点

智能数据库降级：程序启动时优先尝试连接 MySQL，若连接失败（如其他电脑无 MySQL 环境），自动降级到 SQLite，无需任何配置即可运行

数据驱动设计：游戏数值由配置文件控制，无需修改代码即可调整平衡性

Chunk 分块渲染：地图采用分块加载策略，只渲染视野内的区块，大幅提升性能



\## 如何运行

1.安装 Visual Studio 2026

2.打开 练手项目1.sln

3.编译运行



\## 项目现状介绍

一．已完成功能

1.详细地图模块

点击郡级区域时，白色线条围绕该郡边界

点击县级区域时，红色线条围绕该县边界

2.主要地图模块

鼠标滚轮实现地图缩放，支持放大与缩小范围控制

鼠标拖拽平移地图，包含地图边界处理

点击任意格子显示该位置详细信息

需将当前角色栏切换为目标角色后，方可使用事件类型、资源列表等功能

按快捷键 1可进攻当前选中格子

定位功能

支持先检索玩家名称，再检索城池名称

自动将地图居中定位至目标城池

3.势力地图模块

展示玩家势力范围

4.查看玩家

输入玩家名称获取其详细信息

5.创建玩家

输入名称并选择颜色，系统随机分配县城归属

6.查看城池库

通过 MySQL 或 SQLite 输出城池信息

点击“归属”或“等级”列可按降序排列

7.交互体验

所有输入框中，退格键（Backspace）和回车键（Enter）均生效

二．当前不足与待改进之处

1.地图数据不完整：目前仅完成了并州区域的部分按钮与地图数据，其他州郡尚未接入

2.大地图白线问题：在详细地图模块中，大地图背景存在白线显示异常，尚未修复

3.数据库配置依赖：MySQL 连接信息为本地硬编码，其他电脑需依赖 SQLite 降级机制运行
